이 대시보드 사용법

오른쪽 위 편집 모드 버튼을 켜면 사이트 안에서 직접 수정 가능. 저장하면 깃허브에 자동 커밋되고 1~2분 뒤 모든 멤버 화면에 반영돼.

1. 처음 한 번 — 토큰 발급 (1분)

  1. github.com → Settings → Developer settings → Personal access tokens → Fine-grained tokens → Generate new token
  2. Resource owner: ameliepgy314, Repository access: Only select repositories → triview
  3. Repository permissions → Contents: Read and write
  4. "Generate token" → 나오는 github_pat_... 토큰을 복사
  5. 대시보드 우측 상단 편집 모드 버튼 → 토큰 붙여넣기 → 끝. 브라우저(localStorage)에만 저장.

2. 편집 모드에서 할 수 있는 것

  • 태스크 — 상태/담당자/마감일 변경, 제목·메모 수정, 추가, 삭제
  • 단계 — 추가, 이름·설명 수정
  • 회의록 — "+ 새 회의록" 버튼 → 모달에 작성 → 자동으로 .md 파일 + 인덱스 갱신
  • 아이디어 — 카드 위에서 직접 수정 / 삭제 / 추가
  • 멤버 — 이름·역할·색상 수정, 추가
  • D-day — 대시보드 탭의 D-day 카드에서 런칭일 변경

3. 동시 편집 충돌

두 명이 같은 파일을 동시에 저장하면 두 번째 사람이 충돌 알림을 받아. 자동으로 최신 데이터를 다시 가져온 뒤 다시 저장하면 돼.

4. 모든 변경은 깃 히스토리에 남음

누가 언제 뭘 바꿨는지 모두 커밋으로 기록돼. commits 페이지에서 확인.

5. 토큰을 잃어버렸거나 다른 PC에서 사용

토큰은 브라우저별로 저장되니 다른 PC에선 다시 입력해야 해. 새 토큰을 발급해도 되고, 기존 토큰을 가져와도 됨. 토큰 분실 시 GitHub Settings에서 즉시 폐기 가능.

6. 토큰 없이 사용

편집 모드를 안 켜면 그냥 읽기만 됨 (보기 권한). 멤버 중 일부는 읽기 전용으로만 써도 무방.

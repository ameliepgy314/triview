회의록

왼쪽에서 회의를 선택하세요.

이 대시보드 사용법

코드를 몰라도 깃허브 웹에서 파일만 수정하면 바로 반영돼.

1. 태스크 / 진척도 업데이트

data/stages.json 의 각 task에서 status 를 todo / doing / done / blocked 중 하나로 바꿔 저장하면 끝이야. 담당자(owner)와 마감일(due: YYYY-MM-DD)도 같은 파일에서 수정해.

2. 회의록 추가

  1. data/meetings/ 안에 2026-05-02-주간회의.md 같은 마크다운 파일을 새로 만들어.
  2. data/meetings.json 의 배열 맨 앞에 한 줄을 추가: 
    { "date": "2026-05-02", "title": "주간 회의", "file": "data/meetings/2026-05-02-주간회의.md", "attendees": ["가영","멤버2"] }

3. 아이디어 추가

data/ideas.json 의 ideas 배열에 항목을 추가. 카테고리는 자유롭게 추가해도 돼 (예: 브랜드, 제품, 마케팅, 패키지, 채널).

4. 멤버 변경

data/team.json 에서 이름/역할/색상을 수정. id 는 stages/ideas에서 owner로 쓰는 키니까 바꾸면 다른 곳도 같이 업데이트해야 해.

5. 배포

리포 Settings → Pages → Deploy from a branch → main / (root) 로 설정하면 1~2분 뒤 https://ameliepgy314.github.io/triview/ 에서 보여.

6. D-day 변경

data/config.json 의 launchDate 수정. (YYYY-MM-DD)
